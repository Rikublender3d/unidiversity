# UniDiversity 要件定義書

## 1. プロジェクト概要

全国の大学におけるダイバーシティ推進（特にLGBTQ関連施策）の取り組みを一元的に閲覧・比較できる公開Webサイト。
大学生や研究者、大学関係者が各大学の施策状況を把握し、施策推進の参考にすることを目的とする。

### 背景

- 大学のダイバーシティ推進状況はバラバラに公開されており、横断的に比較する手段がない
- 比較サイトがあれば、各大学の施策推進を促す効果が期待される
- Wiki的に年1回程度、みんなで更新していく運用を想定

---

## 2. ターゲットユーザー

| ユーザー | ニーズ |
| --- | --- |
| 大学生（特にLGBTQ当事者・Ally） | 進学先・編入先の比較、自大学の改善提案の根拠 |
| 大学教職員・ダイバーシティ推進担当 | 他大学のベストプラクティス参照 |
| 研究者・ジャーナリスト | 施策の全体像把握、分析・報道 |

## 4. 機能要件

### Phase 1（MVP）

#### 4.1 トップページ

- サイト概要の説明
- 大学一覧への導線
- 大学数・最終更新日の表示

#### 4.2 大学一覧ページ

- 全大学のカード形式一覧表示
- キーワード検索（大学名）
- フィルタリング
  - 地域（北海道・東北・関東・中部・近畿・中国四国・九州沖縄）
  - 大学種別（国立・公立・私立）
  - 施策の有無（各カテゴリごと）
- ソート（五十音順・施策充実度）

#### 4.3 大学詳細ページ

- 大学基本情報（名称、所在地、種別、公式サイトURL）
- ダイバーシティ施策の詳細（下記「比較項目」参照）
- 情報ソースのURL・出典
- 最終更新日

#### 4.4 比較項目（各大学のデータ構造）

##### A. LGBTQ施策

| 項目 | 型 |
| --- | --- |
| 通称名の使用 | 対応状況（○/△/×/不明） |
| 同性パートナーに関する制度対応 | 対応状況 |
| ジェンダーニュートラルトイレ | 対応状況 |
| 性別欄の配慮（入学書類等） | 対応状況 |
| SOGIに関する研修の実施 | 対応状況 |
| アウティング防止の方針 | 対応状況 |

##### B. 相談窓口・支援体制

| 項目 | 型 |
| --- | --- |
| 専門相談窓口の設置 | 対応状況 |
| カウンセラーのSOGI対応研修 | 対応状況 |
| ハラスメント相談との連携 | 対応状況 |

##### C. サークル・学生団体

| 項目 | 型 |
| --- | --- |
| LGBTQ関連の公認団体の有無 | 有/無/不明 |
| 団体名・活動概要 | テキスト |

##### D. 宣言・方針

| 項目 | 型 |
| --- | --- |
| ダイバーシティ宣言 | 有/無 |
| 基本方針の公開 | 有/無 |
| 行動計画の策定 | 有/無 |
| 関連するURLリンク | URL |

##### E. その他

| 項目 | 型 |
| --- | --- |
| 特筆すべき取り組み | テキスト |
| 備考 | テキスト |

### Phase 2（将来拡張）

- 複数大学の横並び比較機能
- 施策充実度のスコアリング・ランキング
- 地図ビュー（日本地図上にプロット）
- データ更新リクエスト機能（フォーム送信）
- CSVエクスポート

## 7. 非機能要件

| 項目 | 要件 |
| --- | --- |
| パフォーマンス | ISR/SSGで静的生成。Lighthouse Performance 90+を目標 |
| レスポンシブ | モバイル・タブレット・PCに対応 |
| アクセシビリティ | WCAG 2.1 AA準拠を目標 |
| SEO | メタタグ、OGP、構造化データ対応 |
| 多言語 | Phase 1は日本語のみ。将来的に英語対応を検討 |

---

## 8. 運用方針

- データ更新は年1回程度を想定（年度末〜年度初め）
- CMS上でコミュニティメンバーが編集・追加
- 情報の正確性は出典URLの明記で担保
- GitHub上でのIssue/PRベースでのフィードバック受付
